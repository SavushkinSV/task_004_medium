1. Проверить код в github Actions, коммит должен иметь статус с синей галочкой.
2. Проверить, что файлы или бинарники называются в соотвествии с заданием.
3. Проверить названия функций и переменных. Названия должны быть понятны и осмыслены. Плохой пример int a = 5; хороший пример int beer_count = 5;
4. Проверить, что функция main не содержит логический блоков и операций ввода/вывода.
5. Проверить, что код не содержит функций, с количеством строк больше 50. 
6. Проверить, что код разделен на логические блоки и соответствует принципам структурного программирования.
